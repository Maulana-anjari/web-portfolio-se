# Portfolio Review

## Context

Tujuan utama website portfolio ini adalah untuk "menjual" profil Maulana Anjari agar lebih mudah dipercaya, dihubungi, dan dipertimbangkan untuk kerja sama proyek, freelance, kolaborasi, atau role di perusahaan.

Review ini menilai website dari sudut pandang hiring manager, potential client, collaborator, dan technical reviewer.

## Penilaian Saat Ini

| Area | Nilai | Catatan |
| --- | ---: | --- |
| Technical quality | 8.8/10 | Lighthouse terbaru sangat solid: Performance 95, Accessibility 88, Best Practices 100, SEO 100. |
| Visual impression | 8.5/10 | Terasa premium, developer-oriented, dan punya personality kuat. |
| Conversion / hiring clarity | 7/10 | Sudah meyakinkan secara visual, tetapi belum selalu langsung menjawab kenapa Maulana harus di-hire dan untuk problem apa. |

Secara umum, portfolio ini sudah berada di level serius dan memorable. Untuk developer portfolio Indonesia, kualitasnya sudah di atas rata-rata. Arah peningkatan berikutnya sebaiknya bukan menambah efek visual, tetapi memperkuat clarity, proof, dan conversion path.

## Kritik Utama

### 1. Terlalu Banyak Aesthetic Confidence, Belum Cukup Business Confidence

Website sudah terlihat keren dan teknikal. Orang teknis kemungkinan akan appreciate visual, stack, dan struktur kontennya.

Namun, hiring manager atau calon client biasanya mencari sinyal cepat:

- Pernah menyelesaikan masalah apa?
- Impact-nya apa?
- Cocok untuk role atau proyek seperti apa?
- Apa bukti bahwa Maulana bisa deliver?

Saat ini, beberapa jawaban itu sudah ada, tetapi belum selalu ditonjolkan sebagai pesan utama.

### 2. Hero Perlu Lebih Tajam

Hero sudah punya visual dan personality kuat, tetapi value proposition bisa dibuat lebih direct.

Contoh positioning yang lebih jelas:

> Backend Engineer for scalable APIs, blockchain systems, and LLM infrastructure.

Subcopy bisa langsung menyebut hasil atau kredibilitas:

> I build production-ready backend systems, Web3 infrastructure, and AI workflows with experience across Pertamina, Sumbu Labs, and research-driven engineering projects.

Tujuannya: dalam 5 detik pertama, visitor langsung paham siapa Maulana, spesialisasinya apa, dan kenapa credible.

### 3. Project Section Perlu Lebih Case-Study Oriented

Project yang ditampilkan sudah bagus, tetapi bisa lebih kuat jika tiap project diformat seperti mini case study.

Struktur yang direkomendasikan:

- Problem
- Role
- Stack
- What I Built
- Result / Metric

Angka seperti berikut harus dibuat sangat terlihat:

- Reduced median issue-to-merge lead time by 51.9%
- Reduced vehicle inspection turnaround by 50%
- Maintained 100% uptime

Metric seperti ini adalah proof yang lebih kuat daripada sekadar daftar teknologi.

### 4. CTA Masih Terlalu Generik

CTA seperti "Start Project" sudah bagus, tetapi untuk portfolio yang tujuannya hiring dan kerja sama, jalur aksi sebaiknya lebih eksplisit.

CTA yang bisa dipertimbangkan:

- Hire Me
- View Resume
- See Case Studies
- Contact for Backend Project

Visitor yang berbeda punya niat berbeda. Hiring manager mungkin ingin resume, client mungkin ingin kontak proyek, sedangkan engineer/collaborator mungkin ingin melihat case study atau GitHub.

### 5. Segmentasi Audience Belum Eksplisit

Website ini menargetkan beberapa audience:

- Perusahaan yang mencari backend engineer
- Client yang butuh proyek backend/Web3/AI
- Collaborator atau founder yang mencari technical partner
- Recruiter atau hiring manager

Masing-masing audience butuh bukti berbeda. Akan lebih kuat jika ada section singkat seperti:

> I'm a fit for:
>
> - Backend Engineer roles
> - Web3/backend infrastructure projects
> - LLM/RAG infrastructure prototypes
> - Technical co-founder or collaborator work

## Rekomendasi Prioritas

### 1. Perkuat Headline dan CTA Hero

Hero harus menjawab tiga hal dengan cepat:

- Siapa Maulana?
- Problem apa yang bisa dia solve?
- Apa langkah berikutnya untuk menghubungi atau mengevaluasi dia?

Rekomendasi CTA hero:

- Primary: Hire Me / Contact Me
- Secondary: View Resume
- Tertiary: See Case Studies

### 2. Tambahkan Proof Strip Setelah Hero

Tambahkan bar pendek setelah hero yang menampilkan sinyal kredibilitas cepat.

Contoh:

> Pertamina EP · Sumbu Labs · UGM · Cardano · FastAPI · Node.js · PostgreSQL

Atau versi impact:

> 51.9% faster issue-to-merge · 50% faster inspection turnaround · Backend/Web3/LLM systems

Proof strip membantu visitor menangkap kredibilitas tanpa harus scroll terlalu jauh.

### 3. Ubah Project Cards Menjadi Mini Case Studies

Project cards sebaiknya tidak hanya menampilkan nama, gambar, dan stack. Tambahkan outcome.

Contoh format:

```text
SumbuPay
Role: Backend Engineer & Co-Founder
Built: Web3 payment backend, QRIS flow, wallet infrastructure
Stack: Node.js, PostgreSQL, Blockchain
Impact: Built core payment infrastructure for production roadmap
```

Untuk project yang punya metric, metric harus tampil lebih dominan daripada stack.

### 4. Tambahkan Section "How I Can Help"

Section services saat ini bisa diperkuat dengan outcome-oriented copy.

Rekomendasi kategori:

- Build production-ready backend APIs
- Design database and system architecture
- Integrate LLM/RAG workflows
- Prototype blockchain-backed systems
- Optimize backend performance and reliability

Gunakan bahasa yang lebih dekat ke kebutuhan client/perusahaan, bukan hanya daftar kemampuan teknis.

### 5. Perkuat Resume dan Contact Path

Pastikan user bisa dengan mudah:

- Download resume
- Email langsung
- Buka LinkedIn
- Buka GitHub
- Melihat project/case study utama

Untuk CTA email, subject bisa dibuat spesifik:

```text
Project Inquiry
Backend Engineer Opportunity
Collaboration Inquiry
```

### 6. Naikkan Accessibility ke 95+

Accessibility sudah naik, tetapi masih ada ruang kecil:

- Pastikan semua button punya accessible name
- Pastikan decorative text tidak dihitung sebagai konten utama
- Pastikan contrast text kecil aman
- Pastikan image utama sudah optimized dan responsive

Menaikkan Accessibility dari 88 ke 95+ akan memperkuat kesan profesional dan matang.

## Recommended Messaging Direction

Website ini sebaiknya berbicara dengan tone:

- Confident, bukan berlebihan
- Technical, tapi tetap business-aware
- Specific, bukan generic
- Outcome-oriented, bukan hanya stack-oriented

Contoh headline direction:

```text
Backend Engineer for scalable APIs, blockchain systems, and LLM infrastructure.
```

Contoh subheadline:

```text
I build production-ready backend systems and technical infrastructure for teams that need reliable APIs, data workflows, and emerging technology prototypes.
```

Contoh proof copy:

```text
Experienced across Pertamina EP, Sumbu Labs, and research-backed blockchain engineering.
```

## Kesimpulan

Portfolio ini sudah berhasil mengatakan:

> Saya teknikal, serius, dan punya taste visual.

Langkah berikutnya adalah membuat website ini lebih keras mengatakan:

> Ini problem yang bisa saya solve untuk kamu, ini bukti saya bisa deliver, dan ini cara paling mudah untuk mulai bekerja sama.

Jika ditingkatkan dari sisi clarity, proof, dan conversion path, portfolio ini bisa menjadi alat hiring dan client acquisition yang jauh lebih kuat.
